# Code of Conduct

## Our Standard

We want this project to stay useful, respectful, and practical for maintainers, contributors, and operators.

Examples of behavior that help:

- being respectful in technical disagreement
- sharing clear reproduction steps and evidence
- assuming good intent while still being honest about problems
- keeping feedback focused on code, behavior, and outcomes

Examples of behavior that are not acceptable:

- personal attacks or insults
- harassment, intimidation, or repeated hostility
- deliberately disruptive behavior
- sharing private credentials, exploit details, or sensitive downstream information in public issue threads

## Scope

This applies in:

- issues
- pull requests
- discussions
- review comments

## Reporting

If you experience or observe unacceptable behavior, contact the maintainers privately when possible.

If the report is also a security issue, follow [SECURITY.md](SECURITY.md) instead of posting sensitive details publicly.
